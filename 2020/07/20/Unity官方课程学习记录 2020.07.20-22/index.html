Day 8 水果忍者原型实现

今天开发的是一个水果忍者游戏的原型，马上这个系列的课就学完了，终于要开始下一阶段的学习了！

游戏实现

  • 一个简易的开始菜单，可以选择难度
  • 一个简易的Game Over菜单
  • 会从屏幕下方弹出水果和炸弹，点到水果会加分，点到炸弹会扣分
  • 当水果没有被点到且掉出下方边界时候，游戏结束

实现思路

  • 通过创建SpawnManager来控制随机物品的生成
  • 设置游戏选择难度按钮，选择难度后SpawnManager开始生成
  • 通过设置一条触发器来判断游戏结束
  • 游戏结束时候会弹出GameOver的信息以及restart按钮

GameManager

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using TMPro;
using UnityEngine.SceneManagement;

public class GameManager : MonoBehaviour
{
    public List targetPrefebs; //list列表 可以用add，remove方法添加或删除
	  //public GameObject[] target;          //数组 占用固定的内存无法添加
    private float speed = 10.0f;
    private float spawnRate = 1.0f;
    public bool gameStarted;
    public TextMeshProUGUI scoreText;
    public GameObject gameOver;
    public GameObject mainMenu;
    private int score = 0;

    IEnumerator SpawnTarget()
    {
        while (gameStarted)
        {
            yield return new WaitForSeconds(spawnRate);
            int index = Random.Range(0, targetPrefebs.Count);
            Instantiate(targetPrefebs[index]);
        }
    }

    public void UpdateSource(int scoreToAdd)
    {
        score += scoreToAdd;
        scoreText.text = "Score:" + score;
    }

    public void GameIsOver() {
        gameStarted = false;
        gameOver.SetActive(true);
    }

    public void RestartGame() 
    {
        SceneManager.LoadScene(SceneManager.GetActiveScene().name);
    }
    public void StartNewGame(int switchD) {
        spawnRate /= switchD;
        gameStarted = true;
        StartCoroutine(SpawnTarget());
        mainMenu.SetActive(false);
    }
}

数组与ArrayList与List

  • 数组
    • 内存中是连续存储的，所以它的索引速度非常快，而且赋值与修改元素也很简单
    • 不足：
      • 需要指定长度，当长度过长时会造成内存浪费
      • 如果不确定长度的时候也会很麻烦
  • ArrayList
    • 命名空间System.Collections下的一部分，在使用该类时必须进行引用
    • 继承了IList接口，提供了数据存储和检索
    • ArrayList对象的大小是按照其中存储的数据来动态扩充与收缩的，不需要指定长度
    • 不足：
      • ArrayList不是类型安全的
      • 存储或检索值类型时通常发生装箱和取消装箱操作，带来很大的性能耗损
  • List
    • List类是ArrayList类的泛型等效类
    • 大部分用法都与ArrayList相似，因为List类也继承了IList接口
    • 在声明List集合时，我们同时需要为其声明List集合内数据的对象类型。

Target

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class Target : MonoBehaviour
{
    private Rigidbody rb;
    GameManager gameManager;

    public int point;
    public float miniSpeed = 12;
    public float maxSpeed = 16;
    public float maxTorque = 10;
    public float xRange = 4;

    public ParticleSystem explosionParticle;
    void Start()
    {
        gameManager = FindObjectOfType();
        rb = GetComponent();
        rb.AddForce(RandomSpeed(), ForceMode.Impulse);
        rb.AddTorque(RandomTorque(), RandomTorque(), RandomTorque());
        transform.position = RandomSpawnPos();
    }

    public void OnTriggerEnter(Collider other)
    {
        Destroy(gameObject);
        if (!gameObject.CompareTag("Bad"))
        {
            gameManager.GameIsOver();
        }      
    }

    void OnMouseDown()
    {
        if (gameManager.gameStarted)
        {
            Instantiate(explosionParticle, transform.position, explosionParticle.transform.rotation);
            gameManager.UpdateSource(point);
            Destroy(gameObject);
        }

        
    }

    Vector3 RandomSpeed()
    {
        return Vector3.up * Random.Range(miniSpeed, maxSpeed); // 因为乘了vector3.up 所以有个初始值
    }

    Vector3 RandomSpawnPos()
    {
        return new Vector3(Random.Range(-xRange, xRange), -1f, 0);
    }

    float RandomTorque() {
        return Random.Range(-maxTorque, maxTorque);
    }
}

Rigidbody.AddTorque

  • 向刚体添加扭矩
  • 只能应用于处于活动状态的刚体。如果 GameObject 处于非活动状态，则 AddTorque 没有效果
  • 默认将唤醒刚体。如果扭矩大小为零，则不会唤醒刚体
  • 添加prefeb不能一起添加，只能一个一个添加
  • 更改变量类型后要回到面板看看数值有没有丢失
